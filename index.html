Teaching Adjustment Plan

Data-Driven

Based on the 15-student assessment data from this grading period, the following adjustments are proposed for the next semester. Each strategy is directly justified by specific learner attainment data.

Finding 1 — Three Students Below 75%: Critical Intervention Required

Felix (67%) · Jack (70%) · Cara (72%)

Data Interpretation: Felix, Jack, and Cara scored below the passing threshold. Their descriptors indicate difficulty with both conceptual understanding and hands-on application.

Teaching Adjustment: Implement a tiered remediation programme. These students will be paired with high-performing peers (Mia, Ben, Iris) for weekly co-troubleshooting lab sessions, supported by simplified diagnostic checklists to reduce cognitive load.

Peer Mentoring + Scaffolded Checklists

Finding 2 — Five Students Clustered in the C Range (70–79%)

Hugo (76%) · Liam (78%) · Noah (74%) · Jack (70%) · Cara (72%)

Data Interpretation: Descriptors such as "Needs reflection" and "Needs consistency" suggest inconsistent effort and limited metacognitive awareness, rather than a lack of fundamental capability.

Teaching Adjustment: Introduce weekly reflective journals where students self-assess against a rubric, and implement fortnightly low-stakes formative quizzes to provide early feedback before summative assessments.

Reflective Journals + Formative Quizzes

Finding 3 — Four A-Range Students: Sustain and Extend Excellence

Mia (96%) · Ben (95%) · Iris (92%) · Ella (90%)

Data Interpretation: Four students achieved A-range results with descriptors like "Outstanding initiative" and "Excellent technical skills." These learners risk disengagement without additional academic challenge.

Teaching Adjustment: Provide extension tasks including leading mini-demonstrations and preparing peer tutorial guides. Advanced ICT scenarios (e.g., network security simulations) will be added to maintain engagement.

Extension Tasks + Peer Teaching Roles

Finding 4 — Network Configuration Is the Lowest-Scoring Competency (76%)

ICT Area: Networking — lowest class average of all 6 competencies

Data Interpretation: Network Configuration scored the lowest among all six tracked competency areas, suggesting the current instructional approach — primarily lecture-based — is insufficient for this topic.

Teaching Adjustment: Redesign the networking unit around a dedicated Lab Immersion Week using Cisco Packet Tracer simulations. Students will physically configure routers, assign IP addresses, and troubleshoot connectivity.

Lab Immersion Week + Packet Tracer Simulations

Finding 5 — Assessment Methods Lack Variety for Practical Learners

Non-test Data: Participation observations and portfolio entries

Data Interpretation: Students who excel in lab work (e.g., Grace — "Good participation") scored lower on written assessments, indicating the grading system does not fully capture hands-on ICT competence.

Teaching Adjustment: Shift toward differentiated assessment by increasing the weight of performance tasks and portfolio evidence. Rubric-based practical assessments will be introduced for every major lab activity.

Differentiated Assessment + Weighted Practical Tasks

Hardware

Printer Troubleshooting

Diagnosed a printer connection issue in the school computer laboratory. The printer was not being detected by Windows 11. Steps taken:

  1. Verified all physical cable connections (USB and power)
  2. Opened Device Manager and identified the missing driver
  3. Downloaded the official driver from the manufacturer website
  4. Cleared the print queue via Windows Services
  5. Ran a printer test page — successfully printed
Outcome: Printer fully restored. Resolution time: 25 minutes. Students impacted: 15.
Networking

Wi-Fi Connectivity Issue

Resolved a classroom-wide connectivity issue where multiple student devices failed to connect to the school network.

  1. Pinged gateway to confirm router was reachable
  2. Identified IP conflict via ipconfig /all
  3. Released and renewed IP addresses using ipconfig /release /renew
  4. Updated wireless adapter drivers on affected machines
  5. Reconfigured DHCP settings on the access point
Outcome: Full connectivity restored for 15 devices. Resolution time: 40 minutes.
System Recovery

Computer Boot Failure

A school computer failed to boot after an incomplete Windows update, displaying a "Boot Configuration Data" error.

  1. Booted from Windows Recovery USB drive
  2. Ran Startup Repair from Advanced Options
  3. Executed bootrec /fixmbr and bootrec /rebuildbcd
  4. Used System Restore to roll back to a stable checkpoint
  5. Verified all system files with sfc /scannow
Outcome: System fully recovered. Data preserved. Resolution time: 1 hour.
Display Hardware

Projector Display Issue

The classroom projector failed to display the teacher's laptop screen during a demonstration lesson.

  1. Tested multiple HDMI cables — found one with bent pins
  2. Attempted alternate HDMI port on both projector and laptop
  3. Switched to VGA adapter as a backup solution
  4. Adjusted display settings: Win+P → Duplicate/Extend
  5. Documented the issue and recommended cable replacement
Outcome: Lesson resumed within 10 minutes. New HDMI cable procurement requested.
Security

Account & Password Recovery

Three students lost access to their school Google accounts. Assisted with recovery and implemented security improvements.

  1. Guided students through Google Account Recovery process
  2. Verified identity using backup email and phone
  3. Enabled 2-Factor Authentication (2FA) on all accounts
  4. Introduced Bitwarden as a secure password manager
  5. Delivered a class-wide password hygiene mini-lesson
Outcome: All accounts recovered. Full class completed security awareness session. Time: 35 minutes.
Software

Software Installation Lab

Led a structured class-wide installation of essential productivity and security software across 15 laboratory computers.

  1. Prepared USB drives with offline installation files
  2. Installed Microsoft Office 365 with valid school licenses
  3. Configured Malwarebytes for real-time protection
  4. Set up LibreOffice as an open-source alternative
  5. Documented a complete software inventory for the lab
Outcome: 15 computers fully configured in under 2 hours. Software inventory documented and filed.
