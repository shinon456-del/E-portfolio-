Teaching Adjustment Plan

Data-Driven

Based on the 15-student assessment data from this grading period, the following adjustments are proposed for the next semester. Each strategy is directly justified by specific learner attainment data.

⚠️

Finding 1: Three Students Below 75 — Critical Intervention Needed

Students: Felix (67%), Jack (70%), Cara (72%)

Data Interpretation: Felix, Jack, and Cara scored below the passing threshold of 75%. Their performance descriptors ("Needs support," "Needs troubleshooting practice," "Needs improvement") indicate difficulty with both conceptual understanding and hands-on application.

Teaching Adjustment: Implement a tiered remediation programme at the start of the next semester. These three students will be paired with high-performing peers (Mia, Ben, Iris) for weekly co-troubleshooting lab sessions. Additionally, simplified diagnostic checklists will be introduced to reduce cognitive load during hardware tasks.

📌 Peer Mentoring + Scaffolded Checklists
📊

Finding 2: Five C-Range Students Clustered at 70–79%

Students: Hugo (76%), Liam (78%), Noah (74%), Jack (70%), Cara (72%)

Data Interpretation: A cluster of five students sit in the C range. Their descriptors ("Average performance," "Needs reflection," "Needs consistency") suggest inconsistent effort and limited metacognitive awareness rather than a lack of capability.

Teaching Adjustment: Introduce weekly reflective journals where students self-assess their own ICT task performance against a rubric. This builds metacognition and accountability. Additionally, low-stakes formative quizzes every two weeks will provide early feedback before summative assessments.

📌 Reflective Journals + Formative Quizzes
🏆

Finding 3: Four A-Range Students — Sustain and Extend Excellence

Students: Mia (96%), Ben (95%), Iris (92%), Ella (90%)

Data Interpretation: Four students consistently achieved A-range scores with descriptors like "Outstanding initiative" and "Excellent technical skills." These learners are at risk of disengagement if the curriculum does not challenge them further.

Teaching Adjustment: Provide extension tasks — such as leading mini-demonstrations, preparing how-to guides for classmates, or tackling advanced ICT scenarios (e.g., network security simulations). This sustains their engagement while benefiting the whole class through peer teaching.

📌 Extension Tasks + Peer Teaching Roles
📈

Finding 4: Network Configuration Scored Lowest (76% Class Avg)

ICT Competency Area: Networking — lowest among all 6 areas

Data Interpretation: Among all six ICT competency areas tracked, Network Configuration had the lowest class average at 76%. This suggests the current instructional approach for networking topics is insufficient — likely due to limited hands-on lab time and abstract delivery.

Teaching Adjustment: Redesign the networking unit to include a dedicated Lab Immersion Week with simulation tools (e.g., Cisco Packet Tracer). Activities will require students to physically configure routers, assign IP addresses, and troubleshoot connectivity — moving from lecture-based to fully experiential learning.

📌 Lab Immersion Week + Packet Tracer Simulations
🎯

Finding 5: Assessment Method Lacks Variety

Non-test Data: Participation observations and portfolio entries

Data Interpretation: Several strong performers in practical labs (e.g., Grace — "Good participation") scored lower on written assessments. This indicates the current grading system may not fully capture practical ICT competence, disadvantaging hands-on learners.

Teaching Adjustment: Shift toward differentiated assessment by increasing the weight of performance tasks and portfolio evidence in the final grade computation. Introduce rubric-based practical assessments for every major lab activity so that hands-on skills are formally credited alongside written tests.

📌 Differentiated Assessment + Weighted Practical Tasks

Printer Troubleshooting

Hardware

Diagnosed a printer connection issue in the school computer laboratory. The printer was not being detected by Windows 11. Steps taken:

  1. Verified physical cable connections (USB and power)
  2. Opened Device Manager → identified missing driver
  3. Downloaded official driver from manufacturer website
  4. Cleared the print queue via Windows Services
  5. Ran printer test page — successfully printed
Outcome: Printer fully restored. Time: 25 minutes. Students impacted: 15.
📶

Wi-Fi Connectivity Issue

Networking

Resolved a classroom-wide Wi-Fi connectivity issue where multiple student devices failed to connect to the school network.

  1. Pinged gateway to confirm router was reachable
  2. Identified IP conflict via ipconfig /all
  3. Released and renewed IP addresses (ipconfig /release /renew)
  4. Updated wireless adapter drivers on affected machines
  5. Reconfigured DHCP settings on the access point
Outcome: Full connectivity restored for 15 devices. Time: 40 minutes.
💻

Computer Boot Failure

System Recovery

A school computer failed to boot after an incomplete Windows update. The system displayed a "Boot Configuration Data" error.

  1. Booted from Windows Recovery USB
  2. Ran Startup Repair from Advanced Options
  3. Executed bootrec /fixmbr and bootrec /rebuildbcd
  4. Used System Restore to roll back to a stable checkpoint
  5. Verified all system files with sfc /scannow
Outcome: System fully recovered. Data preserved. Time: 1 hour.
📽️

Projector Display Issue

Display Hardware

The classroom projector failed to display the teacher's laptop screen during a demonstration lesson.

  1. Tested multiple HDMI cables — found one with bent pins
  2. Attempted alternate HDMI port on both projector and laptop
  3. Switched to VGA adapter as backup solution
  4. Adjusted display settings: Win+P → Duplicate/Extend
  5. Documented issue and recommended cable replacement
Outcome: Lesson resumed in 10 minutes using VGA adapter. Requested new HDMI cable procurement.
🔒

Account & Password Recovery

Security

Three students lost access to their school Google accounts. Assisted with secure account recovery and set up security measures.

  1. Guided students through Google Account Recovery process
  2. Verified identity using backup email/phone
  3. Enabled 2-Factor Authentication (2FA) on all accounts
  4. Introduced password manager (Bitwarden) for secure storage
  5. Conducted mini-lesson on password hygiene for the class
Outcome: All accounts recovered. Class completed security awareness session. Time: 35 minutes.

Software

  3. Configured Malwarebytes for real-time protection

  5. Documented software inventory for the lab
